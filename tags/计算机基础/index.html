CPU的结构和功能

7.1 CPU的结构

7.1.1 CPU的功能

CPU 由运算器和控制器组成

控制器负责控制执行程序的指令序列，运算器负责数据加工，具体功能包括：

  • 指令控制：完成取指令、分析指令和执行指令的操作，即顺序控制
  • 操作控制：产生完成每条指令所需的控制命令
  • 时间控制：给每条指令按时间顺序提供应有控制信号
  • 数据加工：对数据进行算数和逻辑运算
  • 中断处理：处理异常情况和特殊请求

7.1.3 CPU的寄存器

  1. 用户可见寄存器
    • 通用寄存器：可存放操作数，也可作为满足某寻址方式所需的寄存器（如基址寄存器）
    • 数据寄存器：存放操作数，其位数能满足大多数数据类型的数值范围
    • 地址寄存器：可存放地址，也具有通用性，位数必须足够长以最大满足寻址范围
    • 条件码寄存器：根据运算结果由硬件设置（如正负零或溢出）
  2. 控制和状态寄存器：
    • MAR / MDR / PC / IR / 程序状态字 PSW / 中断标记寄存器

7.2 指令周期

7.2.1 指令周期的基本概念

CPU 取出并执行一条指令所需的全部时间称为指令周期

7.2.2 指令周期的数据流

  1. 取指周期：根据 PC 地址从主存中取出指令代码并放入 IR

    1. PC → MAR → 地址总线 → 存储器
    2. CU读命令 → 控制总线 → 主存
    3. 存储器 → 数据总线 → MDR → IR
    4. CU → (PC)+1 → PC

  2. 间址周期：取操作数有效地址

    1. Ad(IR) 或 MDR → MAR → 地址总线 → 存储器
    2. CU读命令 → 控制总线 →存储器
    3. 存储器 → 数据总线 → MDR

  3. 执行周期：不同指令执行周期操作不同，没有统一的数据流

  4. 中断周期：处理中断请求

    1. CU → MAR → 地址总线 → 存储器
    2. CU写命令 → 控制总线 → 存储器
    3. PC → MDR → 数据总线 → 存储器 （保存断点）
    4. CU → PC （形成入口地址）

7.3 指令流水

7.3.1 指令流水原理

取下一条指令的操作和执行当前指令的操作同时进行，实现指令的二级流水

7.3.2 影响流水线性能的因素

  1. 影响执行效率加倍的因素
    • 执行时间一般大于取指时间，需设立缓冲区
    • 遇到条件转移指令时，下一条指令是未知的，采用猜测法
  2. 影响执行效率提高的因素
    • 结构上，不同指令争用同一功能部件而产生资源冲突
      • 操作数和指令分开存储，避免取指和取操作数冲突
    • 数据上，各指令的重叠访存可能改变操作数的读写访问顺序
      • 后退法：将相关指令延迟到操作数被写回寄存器后再执行
      • 定向技术：计算结果不待写入寄存器直接送往所需部件
    • 控制上，转移指令会破坏流水线的连续性，损失大量性能

7.3.3 流水线性能

  • 吞吐率：单位时间内流水线完成的指令数量
  • 加速比：流水线速度与等功能非流水线的速度之比，最大加速比等于流水线段数
  • 效率：流水线各功能段的利用率，常用工作时间时空区与总时空区之比来衡量

7.3.4 流水线多发技术

  • 超标量技术：同时并发多条独立指令
  • 超流水线技术：将时钟周期再分段
  • 超长指令字技术：将多条能并行操作的指令组合

7.4 中断系统

7.4.2 中断请求标记和中断判优逻辑

  1. 中断请求标记：用于确定发出中断请求的中断源

  2. 中断判优逻辑：硬件排队或软件排队

7.4.3 入口地址寻找

  • 硬件向量：①存放无条件转移指令 ②设置向量地址表
  • 软件查询：查询到某中断源发出中断请求后安排转移指令，自动进入中断处理

7.4.4 中断响应

  1. 响应条件：允许中断触发器 EINT，开中断置”1”，关中断置”0”
  2. 响应时间：指令执行周期结束后，统一向所有中断源发中断查询信号

7.4.5 中断屏蔽技术

中断屏蔽技术主要用于多重中断

  1. 多重中断：CPU 在执行中断服务程序过程中，出现了更高优先级的中断请求， 若CPU 暂停现行程序，转而处理新的中断请求，这种中断称为多重中断

  2. 实现多重中断的条件：①提前“开中断” ②可抢夺

  3. 屏蔽技术：设置屏蔽触发器和屏蔽字，例：D > A > C > B

    中断源/屏蔽字 A B C D
    A 1 1 1 0
    B 0 1 0 0
    C 0 1 1 0
    D 1 1 1 1

    响应优先级由硬件设定，处理优先级可以通过屏蔽技术而改变

指令系统

6.1 指令的一般格式

6.1.1 指令的基本格式

一条指令通常包括操作码字段和地址码字段

按照指令中地址码数目的不同，可将指令分成以下几种格式：

  1. 四地址指令：(A1)OP(A2) → A3, A4 指向下一条要执行指令的地址

  2. 三地址指令：(A1)OP(A2) → A3, 用 PC 寄存器代替 A4

​

  1. 两地址指令：(A1)OP(A2) → A3, 将结果存入 A1 或 A2

  2. 单地址指令：OP(A1) → A1, 与 ACC 寄存器进行运算

  3. 零地址指令：只给出操作码，用于堆栈计算机或无操作数指令

利用硬件替代地址码，能有效扩大寻址范围、减少访存次数、缩短指令字长

6.1.2 扩展操作码指令格式

在指令字长有限的情况下，减少地址码以延长操作码，丰富指令种类

  • 不允许短码是长码的前缀
  • 各指令操作码不能重复

6.2 操作数类型和操作类型

6.2.2 数据在存储器中的存放方式

  1. 从任意位置开始，内存利用率高但读写控制复杂
  2. 从每个存储字起始位置开始，一个存取周期完成操作但内存利用率低
  3. 边界对准，即地址的整数倍为起点
    • 32位字长的计算机中，半字是“2”的整数倍，字是“4”的整数倍

6.2.3 操作类型

  • 数据传送/算术逻辑操作/移位/转移/输入输出/其他

6.3 寻址方式

6.3.1 指令寻址

  1. 顺序寻址：由程序计数器 PC 自动形成下一条指令的地址
  2. 跳跃寻址：由转移类指令实现

6.3.2 数据寻址

  1. 立即寻址：形式地址是操作数本身，即立即数
  2. 直接寻址：形式地址是操作数的真实地址，EA = A
  3. 间接寻址：形式地址指向保存操作数有效地址所在存储单元，EA = (A)
  4. 隐含寻址：不显式给出操作数的地址，而在指令中隐含操作数地址
  5. 寄存器寻址：地址码字段直接指出寄存器编号，操作数保存在寄存器内，EA = R
  6. 寄存器间接寻址：寄存器内保存操作数所在存储单元的地址，EA = (R)
  7. 基址寻址：基址由操作系统设定，有效地址等于形式地址与基地址相加，EA = A + (BR)
  8. 变址寻址：变址由用户给出，有效地址等于形式地址与变地址相加，EA = A + (IX)
  9. 相对寻址：有效地址等于形式地址与当前指令地址相加，EA = (PC) + A，常用于转移指令
  10. 堆栈寻址：要求计算机设有堆栈，操作数只能在栈顶存或取，用寄存器保存堆栈指针 SP

6.4 RISC

6.4.1 RISC的主要特征

  • 将复杂指令的功能由使用频率高的简单指令的组合完成
  • 指令长度固定，指令格式种类少，寻址方式指令少
  • 只有存数/取数指令访存，其余指令操作依靠寄存器完成
  • 采用流水线、超标量和超流水线技术，缩短执行时间

6.4.2 RISC相较CISC的优势

  • 充分利用芯片面积
  • 提高计算机运算速度

计算机的运算方法

5.1 无符号数和有符号数

5.1.2 有符号数

有符号数约定最高位为符号位，用“0”表示正号，用“1”表示符号

  1. 原码表示法：符号位表示正负数，数值位即真值的绝对值
  2. 补码表示法：为消除减法运算引入，表示定点整数
  3. 反码表示法：原码和补码运算的过渡
  4. 移码表示法：为解决补码无法比较大小而引入，表示浮点数阶码

5.2 定点数的表示和运算

5.2.1 移位运算

计算机中的小数点是事先约定的，数值左移或右移都以小数点为参照

  1. 算数位移：符号位不变
  1. 逻辑移位：补零即可

5.2.2 加减法运算

5.2.3 溢出判别

  • 一位符号法：当两操作数符号相同，结果与操作数符号不同时，符号溢出
  • 双符号位：最高符号位代表真正的符号

5.2.4 乘法运算

  1. 原码一位乘法，符号位不参与计算
  1. 补码一位乘法，符号参与计算

5.2.5 除法运算

  1. 原码加减交替法

  2. 补码加减交替法

5.3 浮点数的表示和运算

5.3.1 浮点数的表示

阶码位数反映小数点的实际位置，尾数位数反映浮点数的精度

5.3.2 浮点数的规格化

为提高数据精度以及便于比较，将尾数最高位为“1”的浮点数称为规格化数

  • 左规：尾数算术左移一位，阶码减一，可能导致数据丢失，左规可能需要多次
  • 右规：尾数算数右移一位，阶码加一，采用双符号位时尾数溢出时，右规一次

5.3.3 加减法运算

  1. 对阶：求阶差后小阶向大阶看齐
  2. 尾数求和：两个尾数按定点数加减法运算规则进行运算
  3. 规格化：通过左规或右规进行规格化
  4. 舍入：在对阶和规格化时可能导致低位丢失，采用“0舍1入”或“恒置1”法
  5. 溢出：上溢作中断处理，下溢按机器零处理

输入输出系统

4.3 I/O 接口

4.3.1 功能与基本结构

  • 选址功能：分析 CPU 送来的设备码并发出设备选中信号
  • 传送命令：响应 CPU 发出的命令
  • 传送数据：实现主机和 I/O 设备之间的数据传送
  • 反映设备状态：使 CPU 能及时了解各 I/O 设备的工作状态

4.6 DMA 方式

4.6.1 DMA 方式的特点

为有效使用主存，DMA 控制器与 CPU 通常采用以下方式：

  • 停止 CPU 访问主存：当外设需要传送数据时，要求 CPU 放弃使用权
  • DMA 与 CPU 交替访存：将一个 CPU 周期分为两部分，专供两者分别使用
  • 周期挪用：前两种方法的折中，DMA 请求时可能出现以下三种情况
    • CPU 未工作，则 DMA 工作
    • CPU 正工作，则 DMA 等待
    • CPU 正请求，则 DMA 优先，相当于在 CPU 执行指令时插入 DMA 请求

4.6.2 DMA 接口的组成

  • 主存地址寄存器 AR：保存主存中需要交换数据的地址
  • 字计数器 WC：记录传送数据的总字数，每传送一个字计数器减一
  • 数据缓冲寄存器 BR：暂存每次传送的数据
  • 设备地址寄存器 DAR：暂存设备码或寻址信息
  • DMA 控制逻辑与终端机构

4.6.3 DMA 的工作过程

  1. 预处理：指明输入/输出，DAR 送入设备号，AR 送入主存始址，WC 送入字数
  2. 数据传输：以数据块为单位传送
  3. 后处理：DMA 控制器向 CPU 发出中断请求，转入中断服务程序作 DMA 结束处理
特点对比 DMA 程序中断
数据传送 硬件传送 程序传送
响应时间 指令周期内任一存取周期结束时 一条指令执行结束时
异常处理 无 有
保护现场 无 有
优先级 高 低

存储器

3.1 概述

3.1.1 存储器的分类

  1. 按存储介质分类
    1. 半导体存储器，具有易失性
    2. 磁表面/磁芯/光盘存储器，具有非易失性
  2. 按存取方式分类
    1. 随机存储器 RAM：可随机存取任一存储单元的内容，存取时间与物理位置无关
    2. 只读存储器 ROM：派生出 PROM, EPROM, EEPROM, 闪存（U盘）
    3. 串行访问存储器：读写操作必须按照存储单元的物理位置先后寻址
      1. 顺序存取存储器：磁带
      2. 直接存取存储器：磁盘

3.1.2 存储器的层次结构

  • 缓存-主存层次主要解决 CPU 和主存速度不匹配的问题，对程序员透明
  • 主存-辅存层次主要解决存储系统的容量问题，对应用程序员透明（虚拟存储系统）

3.2 主存储器

3.2.2 半导体存储芯片基本结构

  • 数据线数和地址线数共同反映存储芯片的容量
  • 译码驱动将地址线送来的信号翻译为对应存储单元的选择信号
  • 控制线包括读/写控制线和片选线，前者决定操作，后者确定选中的芯片

3.2.3 随机存取存储器

  1. SRAM：利用触发器记忆信息，存取速度快，不需要刷新，常用于Cache

  2. DRAM：利用电容记忆信息，集成度更高，需要刷新，常用于大容量主存系统

    1. 集中刷新：在规定的刷新周期内对存储器所有行进行刷新，存在死区

    2. 分散刷新：将对存储单元的刷新分散到各存取周期中，加长存取周期

    3. 异步刷新：间隔一段时间刷新一行，将刷新安排在译码阶段以避开死区

3.2.5 存储器与CPU的连接

单片存储芯片的容量有限，需要将若干存储芯片连在一起组成足够容量的存储器

  • 位扩展法：仅改变数据线的连接方式，片选线要连接到所有芯片
  • 字扩展法：利用片选信号选中部分芯片
  • 字位同时扩展法

3.2.6 存储器的校验

  • 奇偶校验码：在原编码上加一个校验位。只能检测而不能纠错

    • 配偶即加上校验位后“1”的个数为偶数，配奇同理

  • 汉明码：本质为多重奇偶校验码，具有一位纠错能力

    1. 确定海明码位数：设信息位为 n, 校验位为 k,则有

    2. 确定校验位分布：规定校验位在 0,1,2,4,… 位上，其余为信息位

    3. 分组形成校验关系：第 i 组元素用二进制表示时第 i 位为“1”，形成检测位，如 (1,3,5,7) (2,3,6,7) (4,5,6,7)

    4. 汉明码的纠错过程：检查传送后各组的检测位状态，从而指出错误位置

3.2.7 提高访存速度的措施

  1. 单体多字系统：在一个存取周期内，从同一地址取出多条指令

    ​ 缺点：指令和数据必须在内存中连续存放，否则效果不明显

  2. 多体并行：

    1. 高位交叉编制：按顺序访问存储模块，模块间不能并行访问
    1. 低位交叉编址：采用流水线方式访问各模块，实现并行存取

3.3 高速缓冲存储器

3.3.1 基本工作原理

  • 写操作时必须保证Cache块和主存块的信息同步
    • 写直达法：数据既写入Cache又写入主存，增加访存次数
    • 写回法：数据仅写入Cache，当Cache块被替换时写回主存
  • 其他操作可类比操作系统的页式存储部分

3.3.2 Cache-主存地址映射

  1. 直接映射：每个主存块只能映射到缓存块的唯一位置，若产生冲突则原块被替换

    • 优点：只需对比地址的标记位即可判断所需字块是否在Cache中
    • 缺点：若恰好重复访问缓存相同位置，其他缓存块空闲即使也无法使用

  2. 全相联映射：每个主存块可以映射到缓存块的任意位置

    • 优点：空间利用率高，冲突概率低
    • 缺点：每次查询需要和Cache的全部标记进行对比，成本较高
  1. 组相联映射：每个主存块可以映射到缓存某一组中

系统总线

2.1 总线的基本概念

总线是一组能为多个部件分时共享的公共信息传送线路。

  • 分时即同一时刻只允许一个部件向总线发送信息
  • 共享即总线上可挂接多个部件，多个部件可同时从总线上接收相同的信息
  • 在一个总线周期内传输存储地址连续的多个数据字的总线传输方式，成为猝发传送

2.2 总线分类

2.2.2 系统总线

  • 数据总线：用于传输各功能部件间的数据信息
    • 双向传输总线，位数与机器字长、存储字长有关
  • 地址总线：用于指出数据在主存单元的地址或I/O设备的地址
    • 单向传输总线，位数与存储单元个数有关
  • 控制总线：用于发出各种控制信号

2.2.3 通信总线

按传输方式可分为串行通信和并行通信

  • 串行通信适合远距离传送、并行通信适合近距离数据传输

2.5 总线控制

2.5.1 总线判优控制

常见的几种控制优先权仲裁方式有以下三种

  • 链式查询：所有部件共用一根总线请求线，总线响应线 BG 依次查询请求
    • 优先级：部件离总线控制器越近，其优先级越高，反之越低
    • 优点：只需几根控制线就能实现总线控制，结构简单
    • 缺点：对硬件电路的故障敏感，且优先级不能改变。可能产生“饥饿”

  • 计数器定时查询：当计数值与请求使用总线设备的地址一致时，设备获得总线控制权

    • 优先级：由计数器初值决定查询起点
    • 优点：优先级灵活，对电路故障不如链式查询敏感
    • 缺点：增加了控制线数 log2n +2

  • 独立请求方式：每个设备均有一堆总线请求线和总线允许线，在总线控制器中排队

    • 优先级：由总线控制器决定
    • 优点：响应速度块，优先级灵活
    • 缺点：控制线数量多 2n+1

2.5.2 总线通信控制

  • 同步通信：通信双方由统一时标控制数据传输

    • 优点：传送速度快，控制逻辑简单
    • 缺点：强制性同步，必须按照最慢速度的部件来设计时钟，工作效率低

  • 异步通信：采用应答方式实现控制数据传输，可分为以下三种类型

    • 不互锁：主设备发出请求信号后，不必等待从设备应答，一段时间后撤销请求。

      ​ 从设备收到请求信号后应答，并在一段时间后撤销应答。

    • 半互锁：主设备发出请求信号后，必须等待从设备应答，才能撤销请求。

      ​ 从设备应答后不必等待主设备请求撤销，一段时间后自动撤销应答。

    • 全互锁：主设备发出请求信号后，必须等待从设备应答，才能撤销请求。

      ​ 从设备应答后必须等待主设备请求撤销，才能撤销应答。

  • 半同步通信：结合同步和异步通信的特点，插入等待周期来协调通信双方的配合问题

  • 分离式通信：将一个传输周期分解为两个子周期

    • 第一个子周期中，主设备在获得总线控制权后发出命令、地址等有关信息
    • 第二个子周期中，从设备在收到有关信号后，将主设备需要的数据传输

计算机系统概论

1.1 计算机系统简介

1.1.2 计算机系统的层次结构

层次之间关系紧密，下层是上层的基础，上层是下层的扩展

1.2 计算机的基本组成

1.2.1 冯·诺伊曼计算机

以“存储程序”为基础的各类计算机通称为冯·诺伊曼机

  • 由运算器、存储器、控制器、输入设备和输出设备组成
  • 指令和数据以同等地位存放于存储器内，并可按地址寻访
  • 指令和数据均用二进制数表示
  • 指令由操作码和地址码组成
  • 指令在存储器内按顺序存放
  • 机器以运算器为中心，I/O设备与存储器间的数据传输通过运算器完成

1.2.2 现代计算机

  • 以存储器为中心，I/O操作尽可能绕过CPU，提高系统整体运行效率

1.2.3 计算机功能部件

  • 主存储器：包括存储体、各类逻辑部件及控制电路
    • 存储器地址寄存器 MAR 用于存放欲访问的存储单元地址
      • 其位数对应存储单元个数
    • 存储器数据寄存器 MDR 用于存放来自存储体或即将送往存储体的代码
      • 其位数与存储字长相等
  • 运算器：至少包括三个寄存器和一个算数逻辑单元 ALU
    • 累加器 ACC
    • 乘商寄存器 MQ
    • 操作数寄存器 X
  • 控制器：负责控制指令操作的取指、分析和执行阶段
    • 程序计数器 PC 用于存放当前欲执行指令的地址
      • 具有自动加1的功能 (PC)+1→PC，自动形成下条指令的地址
    • 指令寄存器 IR 用来存放当前的指令
      • 操作码送至 CU，记作 OP(IR)→CU，用于分析指令
      • 地址码送至 MAR，记作 Ad(IR)→MAR，用于数据查找
    • 控制单元 CU 用于分析当前指令所需操作、并发出微操作命令序列
  • I/O：包括各类I/O设备及其对应接口

1.3 计算机的技术指标

1.3.1 机器字长

机器字长指CPU一次能处理数据的位数。字数越长，数的表示范围越大，精度也越高。

1.3.3 运算速度

  • MIPS 即每秒执行多少百万条指令，MIPS = 主频/CPI
  • CPI 即执行一条指令所需的时钟周期
    • 时钟周期 = 1/主频
  • MFLPOS 即每秒执行多少百万次浮点运算 MFLPOS = 操作次数/执行时间*10^6

思考题与习题

  1. 同一个功能在硬件和软件的实现？

​ 硬件和软件是两种完全不同的物质形态，但在逻辑功能上二者是等效的。因此在计算机系统中，同一个功能即可由硬件直接实现，又可在硬件配合下由软件实现。

  1. 计算机如何区分指令和数据？

​ 通过不同时间段区分指令和数据，取指阶段取出的为指令，执行阶段取出的为数据。
