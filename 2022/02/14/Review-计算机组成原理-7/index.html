Review-计算机组成原理-7

CPU的结构和功能

7.1 CPU的结构

7.1.1 CPU的功能

CPU 由运算器和控制器组成

控制器负责控制执行程序的指令序列，运算器负责数据加工，具体功能包括：

  • 指令控制：完成取指令、分析指令和执行指令的操作，即顺序控制
  • 操作控制：产生完成每条指令所需的控制命令
  • 时间控制：给每条指令按时间顺序提供应有控制信号
  • 数据加工：对数据进行算数和逻辑运算
  • 中断处理：处理异常情况和特殊请求

7.1.3 CPU的寄存器

  1. 用户可见寄存器
    • 通用寄存器：可存放操作数，也可作为满足某寻址方式所需的寄存器（如基址寄存器）
    • 数据寄存器：存放操作数，其位数能满足大多数数据类型的数值范围
    • 地址寄存器：可存放地址，也具有通用性，位数必须足够长以最大满足寻址范围
    • 条件码寄存器：根据运算结果由硬件设置（如正负零或溢出）
  2. 控制和状态寄存器：
    • MAR / MDR / PC / IR / 程序状态字 PSW / 中断标记寄存器

7.2 指令周期

7.2.1 指令周期的基本概念

CPU 取出并执行一条指令所需的全部时间称为指令周期

7.2.2 指令周期的数据流

  1. 取指周期：根据 PC 地址从主存中取出指令代码并放入 IR

    1. PC → MAR → 地址总线 → 存储器
    2. CU读命令 → 控制总线 → 主存
    3. 存储器 → 数据总线 → MDR → IR
    4. CU → (PC)+1 → PC

  2. 间址周期：取操作数有效地址

    1. Ad(IR) 或 MDR → MAR → 地址总线 → 存储器
    2. CU读命令 → 控制总线 →存储器
    3. 存储器 → 数据总线 → MDR

  3. 执行周期：不同指令执行周期操作不同，没有统一的数据流

  4. 中断周期：处理中断请求

    1. CU → MAR → 地址总线 → 存储器
    2. CU写命令 → 控制总线 → 存储器
    3. PC → MDR → 数据总线 → 存储器 （保存断点）
    4. CU → PC （形成入口地址）

7.3 指令流水

7.3.1 指令流水原理

取下一条指令的操作和执行当前指令的操作同时进行，实现指令的二级流水

7.3.2 影响流水线性能的因素

  1. 影响执行效率加倍的因素
    • 执行时间一般大于取指时间，需设立缓冲区
    • 遇到条件转移指令时，下一条指令是未知的，采用猜测法
  2. 影响执行效率提高的因素
    • 结构上，不同指令争用同一功能部件而产生资源冲突
      • 操作数和指令分开存储，避免取指和取操作数冲突
    • 数据上，各指令的重叠访存可能改变操作数的读写访问顺序
      • 后退法：将相关指令延迟到操作数被写回寄存器后再执行
      • 定向技术：计算结果不待写入寄存器直接送往所需部件
    • 控制上，转移指令会破坏流水线的连续性，损失大量性能

7.3.3 流水线性能

  • 吞吐率：单位时间内流水线完成的指令数量
  • 加速比：流水线速度与等功能非流水线的速度之比，最大加速比等于流水线段数
  • 效率：流水线各功能段的利用率，常用工作时间时空区与总时空区之比来衡量

7.3.4 流水线多发技术

  • 超标量技术：同时并发多条独立指令
  • 超流水线技术：将时钟周期再分段
  • 超长指令字技术：将多条能并行操作的指令组合

7.4 中断系统

7.4.2 中断请求标记和中断判优逻辑

  1. 中断请求标记：用于确定发出中断请求的中断源

  2. 中断判优逻辑：硬件排队或软件排队

7.4.3 入口地址寻找

  • 硬件向量：①存放无条件转移指令 ②设置向量地址表
  • 软件查询：查询到某中断源发出中断请求后安排转移指令，自动进入中断处理

7.4.4 中断响应

  1. 响应条件：允许中断触发器 EINT，开中断置”1”，关中断置”0”
  2. 响应时间：指令执行周期结束后，统一向所有中断源发中断查询信号

7.4.5 中断屏蔽技术

中断屏蔽技术主要用于多重中断

  1. 多重中断：CPU 在执行中断服务程序过程中，出现了更高优先级的中断请求， 若CPU 暂停现行程序，转而处理新的中断请求，这种中断称为多重中断

  2. 实现多重中断的条件：①提前“开中断” ②可抢夺

  3. 屏蔽技术：设置屏蔽触发器和屏蔽字，例：D > A > C > B

    中断源/屏蔽字 A B C D
    A 1 1 1 0
    B 0 1 0 0
    C 0 1 1 0
    D 1 1 1 1

    响应优先级由硬件设定，处理优先级可以通过屏蔽技术而改变
